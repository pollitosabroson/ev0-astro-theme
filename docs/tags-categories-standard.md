## Reglas generales

1. **Title Case en español**: primera letra de cada palabra en mayúscula, excepto preposiciones cortas (`de`, `del`, `y`, `en`, `a`, `el`, `la`, `los`, `las`, `para`, `con`, `sin`, `por`).
   - ✅ `Fondos de Inversión` &nbsp; ✅ `Educación Financiera`
   - ❌ `fondos de inversión` &nbsp; ❌ `educación financiera` &nbsp; ❌ `FONDOS DE INVERSIÓN`

2. **Tildes obligatorias**: nunca omitas tildes.
   - ✅ `Economía Personal` &nbsp; ✅ `Gestión del Dinero`
   - ❌ `Economia Personal` &nbsp; ❌ `Gestion del Dinero`

3. **Sin CamelCase**: siempre con espacios, nunca junto.
   - ✅ `Blog Personal` &nbsp; ✅ `Disciplina Financiera`
   - ❌ `BlogPersonal` &nbsp; ❌ `DisciplinaFinanciera`

4. **Formato YAML**: usa siempre arrays con corchetes y comillas dobles.
   ```yaml
   categories: ["Finanzas Personales", "Inversiones"]
   tags: ["Ahorro Inteligente", "Fondos Indexados", "Educación Financiera"]
   ```

5. **Cantidad recomendada**:
   - `categories`: 1–3 por artículo
   - `tags`: 3–8 por artículo (evita listas de 20+ tags)

6. **No inventes variantes**: si el concepto ya existe en la lista canónica de abajo, úsalo exactamente igual. Si necesitas uno nuevo, añádelo aquí también.

## Checklist antes de publicar

Antes de hacer merge de un nuevo post, verifica:

- [ ] `categories` — cada valor existe **exactamente** en la lista canónica de este documento
- [ ] `tags` — cada valor existe **exactamente** en la lista canónica de este documento
- [ ] No hay más de 3 categorías ni más de 8 tags
- [ ] Todas las tildes están correctas (`Economía`, `Inversión`, `Planificación`…)
- [ ] Sin CamelCase: `Blog Personal` no `BlogPersonal`
- [ ] El formato es `["Valor Uno", "Valor Dos"]` (array YAML con comillas dobles)
- [ ] No hay duplicados dentro del mismo array
- [ ] Si necesitas un valor nuevo: **añádelo primero a este documento** antes de usarlo
- [ ] `heroImage` sigue el formato `/blogs/nombre-imagen.webp` — siempre bajo `/blogs/`, siempre `.webp`
- [ ] `authors` es un array: `["Alejandro Rosales"]` — no usar `author` en singular

---

## Cómo añadir una nueva categoría o tag

1. Verifica que no existe ya una equivalente en las listas de arriba.
2. Aplica las reglas de Title Case.
3. Añádela a este documento en el lugar alfabético correspondiente.
4. Úsala en el post nuevo.
